STUDENT SIGNUP FLOW:
┌─────────────────────────────────────────────────────────────────────────┐
│ 1. Student visits /signup                                               │
│    ↓                                                                     │
│ 2. Fills form:                                                          │
│    - Name, Email, Phone, Batch Year, Reason, Password, Photo          │
│    ↓                                                                     │
│ 3. System validates & creates signup_request in database               │
│    ↓                                                                     │
│ 4. Verification email sent with token (7-day expiry)                  │
│    ↓                                                                     │
│ 5. Student clicks email link → visits /verify-email?token=...          │
│    ↓                                                                     │
│ 6. Email marked as verified in signup_requests table                   │
│    ↓                                                                     │
│ 7. Admin reviews at /admin/approvals dashboard                        │
│    - Sees student name, email, phone, photo, reason                    │
│    ↓                                                                     │
│ 8. Admin can:                                                           │
│    A) Approve: Set initial password → Creates user account             │
│    B) Reject: Add reason → Signup marked rejected                      │
│    ↓                                                                     │
│ 9. Student can now login with credentials (or re-apply if rejected)   │
│    ↓                                                                     │
│ 10. Dashboard shows profile with uploaded photo                         │
└─────────────────────────────────────────────────────────────────────────┘

ADMIN APPROVAL FLOW:
┌─────────────────────────────────────────────────────────────────────────┐
│ 1. Admin logs in with admin_001 / password123                          │
│    ↓                                                                     │
│ 2. Navigates to /admin/approvals                                       │
│    ↓                                                                     │
│ 3. Sees list of pending signups with:                                 │
│    - Student photo (left side)                                         │
│    - Name, email, phone, batch year                                    │
│    - Why they want to join                                             │
│    - Email verification status (✓ verified or ⏳ pending)             │
│    ↓                                                                     │
│ 4. Only shows students with email_verified = true                     │
│    ↓                                                                     │
│ 5. Click "Approve":                                                     │
│    - Enter initial password                                            │
│    - Creates user account                                              │
│    - Status changed to 'approved'                                      │
│    ↓                                                                     │
│ 6. OR Click "Reject":                                                   │
│    - Enter rejection reason                                            │
│    - Status changed to 'rejected'                                      │
│    - Reason saved for records                                          │
│    ↓                                                                     │
│ 7. Removed from pending list (updated in real-time)                   │
└─────────────────────────────────────────────────────────────────────────┘
